What is Hash and What does it do?

A hash is a fixed-length code generated from data of any size using a mathematical function. It acts like a digital fingerprint, uniquely representing the input data. Hash functions are designed to be fast to compute, produce consistent results for the same input, and be practically impossible to reverse. They're used to verify data integrity, securely store passwords, and efficiently organize information in databases. Even a tiny change in the input produces a significantly different hash, making them useful for detecting alterations in data. Hashes play a crucial role in various cybersecurity applications and data management techniques.

What is it's Role in Security?

Hashing plays a crucial role in security for several reasons:

  • Data Integrity: Hashes ensure data integrity by producing a unique fixed-size string (the hash value) from input data. Even a small change in the input data will produce a significantly different hash. This makes it easy to verify whether data has been altered.

  • Password Storage: Storing passwords in plaintext is a significant security risk. Instead, passwords are hashed before being stored. When a user logs in, the entered password is hashed and compared to the stored hash. This way, even if the password database is compromised, the actual passwords remain secure.

  • Digital Signatures and Certificates: Hashes are used in digital signatures to verify the authenticity and integrity of messages and documents. A digital signature is created by hashing the message and encrypting the hash with the sender’s private key. The recipient can decrypt the hash with the sender's public key and compare it with the hash of the received message to ensure it hasn’t been tampered with.

  • Message Authentication Codes (MACs): Hashes are used in MACs to verify both the integrity and authenticity of a message. A MAC is created by hashing the message along with a secret key. The receiver, who knows the secret key, can perform the same hashing operation and verify that the message has not been altered.

  • Data Deduplication: Hashes help in identifying duplicate data without comparing the actual data itself. This is useful in optimizing storage by eliminating redundant copies of data.

  • Cryptographic Protocols: Hashes are integral to many cryptographic protocols, including SSL/TLS (for secure web communication), blockchain (for creating unique identifiers for blocks of transactions), and many others.

Common Hash Functions:

  • MD5: Widely used in the past but now considered insecure due to vulnerabilities.
  • SHA-1: More secure than MD5 but also considered weak against certain attacks.
  • SHA-256: Part of the SHA-2 family, currently one of the most secure and widely used hash functions.
  • SHA-3: The latest member of the Secure Hash Algorithm family, designed to be highly secure.

This level of security is crucial, even for seemingly simple applications like games, because it helps safeguard user data and prevents unauthorized access to backend systems.

    Hash Examples


    Demonstrating SHA-512 Hashing with Python
    
    import hashlib
    strings = ["test"]
          
    hash_dict = {}
    for string in strings:
        hash_value = hashlib.sha512(string.encode()).hexdigest()
        hash_dict[string] = hash_value
                                            
    for key, val in hash_dict.items():
        print(f"{key:<5}{val:>128}")

    This code is creating a secure digital fingerprint for a piece of text. It starts with a list containing the word (string) "test". The program then uses a special mathematical function called SHA-512 to transform this word into a long string of letters and numbers.

    This transformation is a one-way process, meaning you can't reverse it to get the original word back. The program stores both the original word and its transformed version (the "hash") in a dictionary.

    Finally, in the terminal it prints out the result in a neat format, with the original word on the left and its unique fingerprint on the right. You can also add more strings on consecutive lines to process multiple fingerprints at a time.

    Hashing Individual Characters

    This script demonstrates the hashing of individual characters, adding an additional layer of security to the hash process. When hashing, it's crucial not to re-hash already hashed data, as it would complicate verification and increase computational overhead without enhancing security. Instead, this script hashes each character of the input string independently.

    First, the script initializes a function GetHash that utilizes the SHA-512 algorithm from Python's hashlib library. This function encodes each input character into UTF-8 format, updates a hash object with this encoding, and computes its hexadecimal representation.

    To begin, the user is prompted to input text for hashing using a lambda function called getString, which defaults to "Hello, World!" if no input is provided. The script then iterates through each character in the entered text (stringToHash). For each character, it computes its hash using GetHash. If the computed hash is not already in hashDictionary, it stores this hash as a key with the character as its corresponding value.

    Finally, the script outputs the hashDictionary in a structured format, displaying each hash alongside its corresponding character. Additionally, it saves hashDictionary to a file named 'hash_dictionary.txt'. This file serves as a record for subsequent verification processes, ensuring the integrity and authenticity of the original text through the unique hashes generated.

    
    import hashlib

    def GetHash(input_string, algorithm='sha512'):
        hash_object = hashlib.new(algorithm)
        hash_object.update(input_string.encode('utf-8'))
        hash_hex = hash_object.hexdigest()
        return hash_hex
                                        
    # Function to prompt user for input and return it
    getString = lambda inputString: "Hello, World!" if inputString == "" else inputString
                                        
    # Prompt user for input
    stringToHash = getString(str(input("Enter text to hash: ")))
                                        
    hashDictionary = {}
                                        
    # Hash each character in the input string
    for letterCharacter in stringToHash:
        hashCharacter = GetHash(letterCharacter)
        if hashCharacter not in hashDictionary:
            hashDictionary[hashCharacter] = letterCharacter
                                        
    # Print out the hash dictionary
    print("Hash Dictionary:")
    for key, val in hashDictionary.items():
        print(f"{key:<20}:{val:>2}")
                                        
    # Save hashDictionary to a file for the receiver to use for verification
    with open('hash_dictionary.txt', 'w') as file:
        for key, val in hashDictionary.items():
            file.write(f"{key}:{val}\n")
                                        
    print("Hash dictionary saved to 'hash_dictionary.txt'.")

    Next, the String must be Verified..

    When you send a string along with its hash to someone, they don't "decode" the hash to get the original string (because hashes are one-way functions and cannot be reversed). Instead, they verify the string by computing its hash themselves and comparing it to the hash you provided. If the hashes match, they can be confident that the string you sent is correct. 

    
    import hashlib

    def GetHash(input_string, algorithm='sha512'):
        hash_object = hashlib.new(algorithm)
        hash_object.update(input_string.encode('utf-8'))
        hash_hex = hash_object.hexdigest()
        return hash_hex
    
    # Function to prompt user for input and return it
    getString = lambda inputString: "Hello, World!" if inputString == "" else inputString
    
    # Read hash dictionary from file
    hashDictionary = {}
    with open('hash_dictionary.txt', 'r') as file:
        for line in file:
            line = line.strip()
            if line:
                key, val = line.split(':')
                hashDictionary[key] = val
    
    # Verification process
    print("\nVerification:")
    verificationString = getString(str(input("Enter the same text to verify: ")))
    
    verified = True
    for letterCharacter in verificationString:
        hashCharacter = GetHash(letterCharacter)
        if hashCharacter in hashDictionary and hashDictionary[hashCharacter] == letterCharacter:
            print(f"Hash for '{letterCharacter}' is verified.")
        else:
            print(f"Hash for '{letterCharacter}' is not verified.")
            verified = False
    
    if verified:
        print("\nAll hashes verified successfully.")
    else:
        print("\nVerification failed.")
    When I run this script, here's what happens:
    First, I'm reading in the 'hash_dictionary.txt' file. This file contains all the hashes I created earlier, each linked to its original character. It's like I'm opening up my own personal codebook.
    Next, the script asks me to type in the text I want to verify. This should be the same text I used when I first created the hashes.

    As I enter each character, the script goes to work:
    • It takes each character and creates a new hash for it, just like I did before.
    • Then it checks this new hash against my "codebook" (the hash dictionary).
    • If the hash matches and corresponds to the right character, I get a message saying that character is verified.
    • If something doesn't match up, I'll know right away because the script tells me that character failed verification.

    I can see this process happening for each character I typed in. It's like watching a security check in real-time!

    At the end, I get a final verdict:
    • If every single character checked out okay, I see a message telling me everything was verified successfully.
    • But if even one character didn't match up, I'll know the verification failed.

    This whole process lets me confirm if my original text has remained unchanged. It's a practical way for me to ensure the integrity of my data, catching even the smallest alterations.
